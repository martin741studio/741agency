# DACH Database Link Building & Prospecting

This skill consists of two main phases:
1.  **Prospecting & Scraping**: Finding high-trust DACH business databases.
2.  **Qualification & Planning**: Creating a safe, executable publishing plan.

## Phase 1: DACH Business Database Prospecting & Scraping

### Purpose
Find and extract high-trust, German-language business listing opportunities for a given niche + city/region, by scraping real business databases (Branchenverzeichnisse, Verbände, Kammern, Lieferantenverzeichnisse, Stadtportale), not “SEO submission sites”.

### Inputs
*   **Niche / industry** (placeholder variable)
*   **City + region** (e.g., city, Landkreis, Bundesland)
*   **Optional**: target service keywords, brand name, competitor names (later)

### Output
A structured prospect list (CSV/Sheet-ready) of link opportunities with:
*   Site URL + domain
*   Listing type (Branchenbuch, Verband, Kammer, Stadtportal, Lieferantenverzeichnis, Projektverzeichnis, Ausschreibungsportal, etc.)
*   City/regional relevance score
*   Industry relevance score
*   Submission path (self-submit / contact form / paid listing / membership)
*   Contact info (email/form URL) if available
*   Notes: requirements, costs, proof needed, moderation time

### Hard Rules: What "business databases" means
Include sources that exist for **real-world business purposes**:
*   Firmen-/Branchenverzeichnisse
*   Handels-/Industrie-/Handwerkskammer & member lists (where publicly accessible)
*   Industry associations (Verbände)
*   Supplier directories (Lieferantenverzeichnisse)
*   Regional city portals and economic development sites
*   Project/reference directories (Bau, Industrie, Dienstleister)
*   Tender / vendor pools (Ausschreibungsportale / Anbieterlisten) where relevant

**Exclude**:
*   “Submit your website” SEO pages
*   Auto-generated Web 2.0 farms
*   Generic spam bookmarking sites
*   Comment spam / forum spam targets

### Discovery Logic (Footprints)
The agent uses German footprints + niche + city to locate opportunities, e.g.:
*   `site:.de "Branchenverzeichnis" +[NICHE] +[CITY]`
*   `site:.at "Firmenverzeichnis" +[NICHE] +[CITY]`
*   `site:.ch "Unternehmensverzeichnis" +[NICHE] +[CITY]`
*   `"[CITY]" "Firma eintragen" +[NICHE]`
*   `"[CITY]" "Mitglied werden" +[NICHE]` (for associations/networks)

### Scraping Behavior
*   Crawl only what’s needed (category pages, “Eintragen/Hinzufügen” pages, contact pages)
*   Extract:
    *   submission URLs (`/eintragen`, `/firma-hinzufuegen`, `/profil-anlegen`, `/mitglied-werden`, `/partner`)
    *   pricing indicators (paid tiers)
    *   required fields (NAP, USt-ID, description, images)
    *   contact method if manual approval

### Scoring / Prioritization Rules
**Rank higher if**:
*   DACH language + DACH domain
*   Clear regional + industry fit
*   Real editorial context (not autogenerated)
*   “Business purpose signals” (imprint, company listings, categories, local focus)

**Rank lower if**:
*   Thin content, random topics, obvious link-selling
*   No imprint/ownership transparency
*   Spam footprints (sitewide “submit link” patterns)

## Phase 2: DACH Link Qualification & Publishing Plan Generator

### Purpose
Turn the scraped prospect list into an actionable link-building plan that is:
*   safe (anchor + velocity control)
*   locally relevant
*   human/agent-executable
*   scalable

### Inputs
*   Prospect list from Phase 1
*   Target website + target pages (homepage + relevant service pages)
*   Niche + city/region
*   Current anchor distribution (or “unknown” → agent assumes conservative)

### Outputs
1.  **Qualified list** (Approve / Reject / Manual Review)
2.  **Link type + action**
    *   Self-submit listing
    *   Manual contact
    *   Membership request
    *   Partner/case study outreach (deferred to outreach agent later)
3.  **Anchor assignment per placement** (with quotas)
4.  **Content requirement per placement** (short bio, long description, images, PDF asset)
5.  **Velocity schedule** (weekly pacing)

### Anchor Strategy (Hard-coded safety rules)
**Target mix**:
*   40–50% Brand
*   20–30% Naked URL
*   10–15% Generic (“Website”, “Mehr erfahren”)
*   5–10% Partial match
*   0–3% Exact match (only on high-trust German sites)

**Hard rule**:
If site is not clearly high-quality and relevant → **exact match is forbidden**.

### Local Relevance Requirements
*   For city pages: prioritize city/regional listings first
*   For niche pages: prioritize industry listings/associations first
*   Maintain language consistency (German descriptions)

### Publishing Plan (Example Logic)
*   **Week 1–2**: core business citations + regional directories (brand/URL anchors)
*   **Week 3–4**: industry directories + supplier lists
*   **Week 5+**: selective outreach targets + partner opportunities

### Anti-pattern Detection (Reject Conditions)
**Reject if**:
*   unrelated niche
*   non-DACH language
*   auto-generated network vibes
*   no real business purpose
*   too many outbound links / obvious paid link farm
